## Why — 为什么要做

### 现状
二狗每次对话都是"失忆"状态——不记得用户说过什么、是什么人、有什么偏好。这让"个人助手"变成了"陌生人助手"，和竞品没有任何差异。

### 期望
二狗能记住用户的关键信息（事实、习惯、性格、意图），在后续对话中自然引用。用户感受到"这个助手真的了解我"。

> **Next项目验证**: 在Next中已验证此方案——SQLite + 记忆分类 + 最近访问排序 + prompt注入，效果满足日常使用。

## What — 做什么

### 变更概述
- [ ] 后端: 记忆CRUD API + 记忆表
- [ ] 后端: prompt构建时注入相关记忆
- [ ] 后端: 记忆内容sanitize防注入
- [ ] 客户端: 记忆管理页面（查看/删除）
- [ ] 客户端: 对话中"帮我记住..."的交互

### 新增能力
- `memory-crud`: 记忆的创建、查询、删除
- `memory-injection`: 对话时自动注入相关记忆到prompt

### 修改能力
- `chat-system`: 对话请求构建时加入记忆上下文
