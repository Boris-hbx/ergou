## 1. 安全检查 ⚠️

> AI产品的安全检查是必经环节，不是可选项。

### Prompt安全
- [ ] 所有用户输入注入prompt前已sanitize（长度截断+控制字符过滤+标签替换）
- [ ] 新增的tool不会泄露其他用户数据
- [ ] 新增的tool不会执行危险操作（删除数据/发送请求等需要确认）
- [ ] System prompt修改后，二狗仍能正确拒绝越狱尝试

### 数据安全
- [ ] 敏感数据加密存储
- [ ] 不该上传的数据没有发送到云端
- [ ] API端点有正确的身份验证
- [ ] 新增的数据表/字段有适当的访问控制

### 对抗性测试（手动）
- [ ] 尝试通过对话让二狗泄露system prompt → 被拒绝
- [ ] 尝试通过对话让二狗执行未授权操作 → 被拒绝
- [ ] 尝试通过对话让二狗暴露其他用户信息 → 被拒绝
- [ ] 输入超长文本 → 被截断，不导致崩溃或异常行为

## 2. 体验检查 🐕

> 确保新功能不破坏二狗的人设。

### 人设一致性
- [ ] 二狗的回复符合"毒舌损友"性格（不是客服、不是导师）
- [ ] 不违反16条行为边界（特别检查: 不主动打断、不情绪化、不卖萌）
- [ ] 语气在不同场景下自然切换（正常/出错/深夜/用户不想干活）

### 记忆一致性（如涉及）
- [ ] 新记忆正确保存
- [ ] 记忆在后续对话中被自然引用（不是逐条念出来）
- [ ] 用户可以查看/删除记忆

### 交互体验
- [ ] 正常流程顺畅（参照 usecases.md 的 Main Success Scenario）
- [ ] 异常情况有合理的降级处理
- [ ] 不让用户等太久（AI响应开始时间 < 3秒）
