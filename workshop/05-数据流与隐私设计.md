## 2. 数据分类与处理策略

| 数据类型 | 敏感等级 | 存储位置 | 加密 | 是否上传云端 |
|---------|---------|---------|------|------------|
| 对话文本 | 中 | 本地SQLite | 是 | 仅当前轮发送给LLM，不持久化在云端 |
| 会话摘要 | 中 | 本地SQLite | 是 | 不上传 |
| 用户偏好/画像 | 中 | 本地SQLite | 是 | 不上传 |
| 普通照片记忆 | 中 | 本地文件 | 是 | 需要理解时发送给多模态LLM，用完即弃 |
| 证件类照片 | **高** | 本地文件 | **强加密** | **绝不上传** |
| OCR提取的文字 | 中 | 本地SQLite | 是 | 不上传（端侧OCR时） |
| App设置 | 低 | DataStore | 否 | 不上传 |
| 崩溃日志 | 低 | Bugly云端 | — | 自动上传（不含用户内容） |
| 使用统计 | 低 | 友盟云端 | — | 自动上传（匿名化） |

## 3. 核心数据原则

| 原则 | 说明 |
|------|------|
| **本地优先** | 所有用户内容数据默认且仅存储在设备本地 |
| **最小上传** | 只有AI推理必需的数据才发送到云端，且不在云端留存 |
| **可删除** | 用户可以删除任何数据，包括"清除所有"一键重置 |
| **透明** | 明确告知用户哪些数据会发送到云端（首次使用引导） |
| **无账号** | MVP阶段无需注册登录，无用户身份数据 |

## 4. 加密方案

| 数据 | 加密方式 | 说明 |
|------|---------|------|
| SQLite数据库 | SQLCipher 或 Room + Android Keystore | 整库加密，密钥由系统安全存储 |
| 普通照片文件 | AES-256 (密钥存Android Keystore) | 文件级加密 |
| 高敏感照片（证件） | AES-256 + 独立PIN/生物识别解锁 | 双重保护 |
| 网络传输 | HTTPS (TLS 1.3) | 所有云端通信强制HTTPS |
| API Key | Android Keystore 存储 | 不硬编码在代码中 |

## 5. AI安全防护（来自Next项目验证）

> 详见 [06-Next项目已验证经验.md](./06-Next项目已验证经验.md)

### 5.1 Prompt注入防护

所有用户生成内容（记忆、人物信息、任务标题等）注入AI prompt前**必须sanitize**：

| 防护 | 说明 |
|------|------|
| 长度截断 | 用户输入截断到合理最大长度，防止上下文溢出 |
| 控制字符过滤 | 去除控制字符（保留换行），防止格式注入 |
| 标签替换 | `<` `>` 替换为空格，防止伪造系统指令标签 |

### 5.2 安全事件记录

```sql
CREATE TABLE security_events (
    id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    event_type TEXT NOT NULL,        -- 'jailbreak', 'data_leak_attempt', 'privilege_escalation'
    severity TEXT NOT NULL,          -- 'low', 'medium', 'high'
    description TEXT,
    conversation_id TEXT,
    admin_notified INTEGER DEFAULT 0,
    created_at TEXT NOT NULL
);
```

### 5.3 三级响应机制

| 级别 | 触发条件 | 二狗的响应 |
|------|---------|-----------|
| Level 1 | 一般试探（改名、角色扮演） | 礼貌拒绝 + 话题重定向 |
| Level 2 | 明确攻击（套数据、越狱） | 正式警告 + 记录安全事件(medium) |
| Level 3 | 持续攻击/高危操作 | 通知管理员 + 上报 + 暂停AI服务 |
