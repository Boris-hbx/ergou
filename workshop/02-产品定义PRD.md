| 维度 | 定义 | 待确认 |

| **产品名称** | 二狗 (Ergou) | 是否需要更正式的品牌名？ |
| **产品定位** | 隐私优先、有温度、越用越懂你的个人AI助手 | 待拍板 |
| **核心价值** | 不是最聪明的AI，而是最懂你的AI | — |

| **目标用户** | 待定（见第2节） | **需讨论** |

## 2. 目标用户画像（待讨论确认）

### 选项A：面向个人用户（To C）

| 维度 | 描述 |
|------|------|
| **年龄** | 20-40岁，智能手机重度用户 |
| **特征** | 信息过载、需要管理日程和碎片信息、注重隐私 |
| **痛点** | 现有AI助手不记得我是谁、不了解我的习惯、数据隐私担忧 |
| **核心需求** | 一个"真正了解我"的助手，帮我记事、提醒、处理日常事务 |

### 选项B：面向公司内部（To B）

| 维度 | 描述 |
|------|------|
| **用户** | 公司员工 |
| **特征** | 办公场景为主、需要企业知识库集成 |
| **痛点** | 企业内部信息检索困难、跨系统操作繁琐 |
| **核心需求** | 企业级AI助手，对接内部系统，提升办公效率 |

### 选项C：先To C验证，后期可扩展To B

> **建议采用选项C**: 先以个人助手定位做MVP验证，产品形态和技术架构成熟后再考虑企业级场景。

## 3. 核心场景定义（TOP 3，建议）

> 个人助手范围极广，MVP必须聚焦。以下是建议的3个核心场景，待讨论确认。

### 场景一：智能对话 — "聊得来"

| 要素 | 说明 |
|------|------|
| **用户故事** | 作为用户，我想和二狗自然对话，问问题、聊想法、求建议 |
| **核心能力** | 多轮对话、上下文理解、知识问答、文本生成 |
| **差异化** | 有性格、有温度、记得你说过的话（记忆系统） |
| **MVP范围** | 文字对话 + 多轮上下文 + 基础记忆 |
| **排除** | 语音对话（MVP后期或V2） |

### 场景二：个人记忆管理 — "帮你记"

| 要素 | 说明 |
|------|------|
| **用户故事** | 作为用户，我想让二狗帮我记住重要的事情，以后随时能问出来 |
| **核心能力** | 对话中自动提取记忆、拍照记录、主动提醒 |
| **差异化** | 不只是笔记工具，是"懂你"的记忆助手 |
| **MVP范围** | 手动记录 + 照片保存+OCR + 关键词检索 |
| **排除** | 自动记忆提取、语义检索（V2）、主动关联回忆（V3） |

### 场景三：日常事务助手 — "替你办"

| 要素 | 说明 |
|------|------|
| **用户故事** | 作为用户，我想让二狗帮我设提醒、查天气、做简单计算和翻译 |
| **核心能力** | 系统集成（提醒/日历）、工具调用（天气/计算/翻译） |
| **差异化** | 结合记忆——"提醒我下周二那个会议之前准备材料" |
| **MVP范围** | 提醒设置 + 天气查询 + 简单工具调用 |
| **排除** | 深度系统集成（联系人/电话/无障碍） → V2 |

## 4. MVP功能边界

### 做什么（MVP In Scope）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 文字对话 | P0 | 核心交互方式 |
| 多轮对话上下文 | P0 | 工作记忆 |
| 会话历史管理 | P0 | 查看/删除历史会话 |
| 基础长期记忆 | P0 | 用户主动保存的信息 + 偏好 |
| 照片记忆 | P1 | 拍照/选图 + OCR文字提取 + 保存 |
| 短期记忆（会话摘要） | P1 | 跨会话上下文衔接 |
| 提醒/闹钟设置 | P1 | 通过对话设置系统提醒 |
| 天气查询 | P2 | 简单工具调用示例 |
| 深色模式 | P2 | 基础体验 |
| 设置页面 | P2 | 模型选择、记忆管理、隐私设置 |

### 不做什么（MVP Out of Scope）

| 功能 | 原因 | 计划阶段 |
|------|------|---------|
| 语音对话 | 增加复杂度，先验证文字交互 | V2 |
| 端侧推理 | MVP先验证产品价值，不追求离线 | V2 |
| 自动记忆提取 | 需要复杂的LLM pipeline | V2 |
| 向量语义检索 | MVP用关键词检索够了 | V2 |
| 联系人/日历深度集成 | 权限和隐私复杂 | V2 |
| 多设备同步 | 一个人一台设备先跑通 | V3 |
| iOS / 鸿蒙 | Android先行 | Phase 2/3 |
| 社交/分享功能 | 不是核心价值 | 视需求 |
| 账号体系/登录 | MVP本地使用，无需账号 | 需要云同步时 |

## 5. 信息架构（App模块结构）

```
二狗 App
├── 对话（首页/主界面）
│   ├── 新对话
│   ├── 历史会话列表
│   └── 对话详情（多轮聊天）
│
├── 记忆
│   ├── 文字记忆列表（按标签/时间）
│   ├── 照片记忆列表（相册式浏览）
│   ├── 搜索记忆
│   └── 记忆详情（查看/编辑/删除）
│
├── 提醒（可选独立Tab，或融入对话）
│   ├── 待办提醒列表
│   └── 已完成列表
│
└── 设置
    ├── AI模型选择/配置
    ├── 记忆管理（查看/清除所有记忆）
    ├── 隐私设置（记忆开关、数据加密）
    ├── 存储管理（照片占用空间）
    ├── 外观（深色模式）
    └── 关于
```
