## Phase 1：Android MVP（第2-9周）

> **目标**: 核心场景可用——对话、记忆、照片记忆、简单工具。能自己日常使用。

### Sprint 1：对话核心（第2-3周）

**目标**: 像ChatGPT一样流畅的对话体验 + 二狗性格

- [ ] **UI: 对话界面**
  - 聊天气泡布局（用户右侧、二狗左侧）
  - 输入框 + 发送按钮
  - 流式打字效果（SSE流式返回）
  - Markdown渲染（代码块、列表、加粗等）
  - 加载状态（"二狗正在想..."）
- [ ] **UI: 会话管理**
  - 新建会话
  - 历史会话列表（侧边栏或单独页面）
  - 删除会话
- [ ] **工作记忆**
  - 内存维护当前对话历史 `List<Message>`
  - 滑动窗口：超过20条时压缩早期消息
- [ ] **数据层**
  - Room数据库设计（会话表 + 消息表）
  - 本地持久化对话历史
- [ ] **架构搭建**
  - MVVM: ChatViewModel → ChatRepository → LLMService
  - LLMService接口 + DeepSeekService实现
  - Koin DI配置

**完成标准**: 能流畅多轮对话，二狗性格鲜明，历史会话可查看。

### Sprint 2：记忆系统（第4-5周）

**目标**: 二狗能记住你说过的重要事情

- [ ] **数据库: 记忆表**
  - 复用Next的`ergou_memories`表设计
  - 字段：id, category(fact/habit/personality/intent), content, importance, repeat_count, emotion_weight, created_at, last_accessed_at, access_count
  - 加密存储（Room + SQLCipher 或 Android Keystore）
- [ ] **数据库: 人物表**
  - 复用Next的`ergou_people`表设计
- [ ] **记忆存储**
  - 用户通过对话主动记忆："帮我记住XXX"
  - LLM判断对话中的重要信息 → 自动提取为记忆（V1先做手动，V2做自动）
  - 记忆重要度评分（LLM打分1-5）
- [ ] **记忆召回**
  - 新对话时，按最近访问时间加载TOP 20条记忆注入prompt
  - 记忆强度模型：重要度 × 重复系数 × 情感权重 × 时间衰减
  - 回忆风格：强记忆→自信提及，弱记忆→"我大概记得..."
- [ ] **上下文引擎**
  - 复用Next的动态prompt组装模式
  - system prompt = 人设 + 人物上下文 + 记忆上下文 + 时间上下文
  - Sanitize所有用户生成内容（防prompt注入）
- [ ] **UI: 记忆管理页面**
  - 记忆列表（按分类/时间）
  - 查看/编辑/删除单条记忆
  - 清除所有记忆

**完成标准**: 说"帮我记住我下周要出差"→ 下次对话二狗能回忆。记忆有自然的模糊感。

### Sprint 3：照片记忆（第5-6周）

**目标**: 拍照或选图让二狗记住

- [ ] **拍照/选图**
  - 相机权限申请
  - 拍照 → 保存到App私有目录
  - 从相册选图
  - 图片压缩（80%质量JPEG）+ 生成缩略图
- [ ] **OCR文字提取**
  - 端侧OCR：Google ML Kit Text Recognition（免费、离线可用）
  - 提取的文字存入记忆表，关联图片路径
- [ ] **图片元信息**
  - SQLite存储：原图路径、缩略图路径、OCR文字、用户备注、标签、时间戳
  - 安全等级标记（普通/敏感）
- [ ] **UI: 照片记忆**
  - 对话中发送图片入口
  - 照片记忆列表（相册式）
  - 照片详情（原图 + OCR文字 + 编辑标签）
  - 存储空间统计
- [ ] **对话中检索照片记忆**
  - "我上次拍的那个名片"→ 关键词匹配OCR文字 → 返回图片

**完成标准**: 拍名片 → OCR提取文字 → 下次问"那个名片上的电话"能找到。

### Sprint 4：工具调用 + 提醒（第6-7周）

**目标**: 二狗能帮你做事，不只是聊天

- [ ] **工具调用框架**
  - 复用Next的Tool Use循环模式（最多5轮）
  - 工具定义接口
  - LLM function calling → 执行工具 → 结果回注
- [ ] **核心工具（MVP精简集）**
  - `set_reminder`: 设置提醒/闹钟（AlarmManager）
  - `query_weather`: 查天气（免费API如和风天气）
  - `save_memory`: 保存记忆
  - `search_memory`: 搜索记忆
  - `get_date_time`: 获取当前日期时间
  - `simple_calculate`: 简单计算
- [ ] **提醒系统**
  - AlarmManager + NotificationManager
  - 提醒列表UI
  - 提醒到时弹通知
- [ ] **Token用量追踪**
  - 每次API调用记录input/output tokens
  - 设置页面显示本月用量和估算成本

**完成标准**: "明天下午3点提醒我开会" → 设好提醒 → 到时弹通知。

### Sprint 5：设置 + 隐私 + 打磨（第7-8周）

  - 隐私设置（记忆功能开关）
  - 存储管理（照片空间）
  - 深色模式
  - 关于页面
- [ ] **隐私与安全**

  - 权限申请说明
  - 数据导出（可选）
  - 清除所有数据
- [ ] **崩溃监控接入**

- [ ] **体验打磨**
  - 启动页（二狗Logo）
  - 空状态页面

## 风险与应对

| 风险 | 应对 |
|------|------|
| DeepSeek API不稳定 | 备选通义千问，LLMService可切换 |
| 二狗人设在DeepSeek上效果不好 | Phase 0充分调试，必要时换模型 |
| 记忆系统复杂度超预期 | MVP只做手动记忆+简单召回，自动提取留给V2 |
| 一个人做太累 | 严格控制MVP范围，能砍就砍，先跑通核心链路 |
| 照片功能开发量大 | 先做最简版（拍照+OCR），智能理解留V2 |
