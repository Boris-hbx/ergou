## 2. 运营阶段成本估算

### 2.1 AI API 调用成本（核心成本项）

以 DeepSeek API 为例（当前定价）:

| 用户行为 | Token估算 | 单次成本 |
|---------|----------|---------|
| 一轮简单对话 | 输入~500 + 输出~300 tokens | ≈ ¥0.001 |
| 一轮复杂对话（含记忆注入） | 输入~2000 + 输出~500 tokens | ≈ ¥0.003 |
| 会话摘要生成 | 输入~3000 + 输出~200 tokens | ≈ ¥0.003 |
| 图片理解（多模态） | 输入~1000 + 图片tokens + 输出~300 | ≈ ¥0.01 |

### 2.2 按用户规模估算月成本

假设每个活跃用户每天平均 20 轮对话 + 2 次图片理解:

| 日活(DAU) | 月对话成本 | 月图片成本 | 月合计 | 年合计 |
|-----------|----------|----------|--------|--------|
| 1（自己用） | ¥2 | ¥1 | **¥3** | ¥36 |
| 10（内测） | ¥20 | ¥6 | **¥26** | ¥312 |
| 100 | ¥200 | ¥60 | **¥260** | ¥3,120 |
| 1,000 | ¥2,000 | ¥600 | **¥2,600** | ¥31,200 |
| 10,000 | ¥20,000 | ¥6,000 | **¥26,000** | ¥312,000 |

> **关键结论**: DeepSeek 极其便宜。自己用和小范围内测几乎不花钱。但用户量过千后成本开始有感，过万需要认真考虑商业模式。

## 3. 成本控制策略

| 策略 | 说明 |
|------|------|
| **按需调用** | 简单问题不调大模型（如纯查天气走API，不走LLM） |
| **上下文压缩** | 会话摘要代替完整历史，减少输入token |
| **端侧OCR优先** | 图片文字识别优先用免费端侧方案，复杂理解才调云端 |
| **用户配额** | 如对外服务，设每日对话次数上限或付费解锁 |
| **缓存** | 常见问题（天气、汇率等）缓存结果，减少重复调用 |
| **模型分级** | 简单任务用小模型/便宜模型，复杂任务才用高级模型 |

## 4. 商业模式

> 现阶段不用急着定，但要有初步思考，影响产品设计决策。

### 4.1 可选商业模式

| 模式 | 说明 | 适合阶段 | 对产品设计的影响 |
|------|------|---------|-----------------|
| **完全免费（自用/展示）** | 自己用+作为技术能力展示 | 当前 | 无限制 |
| **免费+增值（Freemium）** | 基础功能免费，高级功能付费 | 用户量起来后 | 需区分免费/付费功能 |
| **订阅制** | 月/年付费解锁全部功能 | 用户价值验证后 | 需要足够的付费点 |
| **API额度制** | 每天N次免费对话，超出付费 | 成本压力大时 | 需要计数和限流逻辑 |
| **公司内部产品** | 纳入公司产品线，公司承担成本 | 如公司认可 | 需要对接公司体系 |

### 4.2 建议路径

```
当前 ──→ MVP验证 ──→ 内测推广 ──→ 考虑商业化
(自用)    (免费)      (免费)      (Freemium/订阅)
```

**当前不需要做的**: 支付系统、会员体系、广告SDK

**当前需要预留的**: 用户配额统计（API调用次数），为未来限流/付费做数据准备
